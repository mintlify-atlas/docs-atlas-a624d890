The main `hyperstack` crate provides a unified interface to all HyperStack components through feature flags and convenient re-exports.

<Info>
  View on [docs.rs](https://docs.rs/hyperstack)
</Info>

## Installation

```toml
[dependencies]
hyperstack = "0.2"
```

Or with specific features:

```toml
[dependencies]
hyperstack = { version = "0.2", features = ["full"] }
```

## Feature Flags

<ParamField path="interpreter" type="feature" default="enabled">
  AST transformation runtime and VM for processing blockchain events
</ParamField>

<ParamField path="macros" type="feature" default="enabled">
  Procedural macros for defining stream specifications
</ParamField>

<ParamField path="server" type="feature" default="enabled">
  WebSocket server and projection handlers
</ParamField>

<ParamField path="sdk" type="feature">
  Rust client for connecting to HyperStack servers
</ParamField>

<ParamField path="runtime" type="feature">
  Re-exports runtime dependencies for generated code
</ParamField>

## Module Re-exports

The crate conditionally re-exports sub-crates based on enabled features:

```rust
pub use hyperstack_interpreter as interpreter;
pub use hyperstack_macros as macros;
pub use hyperstack_server as server;
pub use hyperstack_sdk as sdk;
```

## Runtime Helpers

When the `runtime` feature is enabled, the crate exposes helper functions for generated code:

```rust
use hyperstack::runtime::serde_helpers::pubkey_base58;

#[derive(Serialize, Deserialize)]
struct MyAccount {
    #[serde(with = "pubkey_base58")]
    owner: [u8; 32],
}
```

### Serde Helpers
