title: hyperstack-server
description: WebSocket server and projection handlers for HyperStack streaming pipelines
---

The `hyperstack-server` crate provides a builder API for creating HyperStack servers that process Solana blockchain data and stream entity updates over WebSockets.

<Info>
  View on [docs.rs](https://docs.rs/hyperstack-server)
</Info>

## Installation

```toml
[dependencies]
hyperstack-server = "0.2"
```

## Quick Start

```rust
use hyperstack_server::{Server, Spec};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    Server::builder()
        .spec(my_spec())
        .websocket()
        .bind("[::]:8877".parse()?)
        .health_monitoring()
        .start()
        .await
}
```

### Specification

<ResponseField name="Spec" type="struct">
  Specification for a HyperStack server containing bytecode and configuration
  
  ```rust
  pub struct Spec {
      pub bytecode: MultiEntityBytecode,
      pub program_ids: Vec<String>,
      pub parser_setup: Option<ParserSetupFn>,
      pub views: Vec<ViewDef>,
  }
  ```
  
  ### Methods
  
  <ParamField path="new" type="fn(bytecode, program_id) -> Self">
    Create a new spec with bytecode and program ID
  </ParamField>
  
  <ParamField path="with_parser_setup" type="fn(ParserSetupFn) -> Self">
    Add custom parser setup function
  </ParamField>
  
  <ParamField path="with_views" type="fn(Vec<ViewDef>) -> Self">
    Add view definitions
  </ParamField>
</ResponseField>
